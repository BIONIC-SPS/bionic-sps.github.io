# BIONIC

A cutting-edge research project pushing the boundaries of biological and computational integration.

## Architecture

The BIONIC website uses a dual architecture:

- **Jekyll** (root directory): Generates the static site served by GitHub Pages.
- **React + Vite** (`frontend/` directory): Builds interactive components (e.g. the network visualisation) whose output is committed to `assets/react/` and included by Jekyll.

> **Note:** The root `package.json` and `src/` directory are from a previous Create React App setup and are **no longer used**. All active React development happens inside `frontend/`.

## Prerequisites

- Ruby and [Bundler](https://bundler.io/) (for Jekyll)
- Node.js (v18 or higher) and npm (for the React frontend)

## Local Development

### Jekyll site (run from the repository root)

1. **Install Ruby dependencies**:
   ```bash
   bundle install
   ```

2. **Start the development server**:
   ```bash
   bundle exec jekyll serve
   ```
   The site will be available at http://localhost:4000

### React frontend (run from the `frontend/` directory)

1. **Install Node dependencies**:
   ```bash
   cd frontend
   npm install
   ```

2. **Build for production** (outputs to `assets/react/`):
   ```bash
   npm run build
   ```
   Commit the updated `assets/react/` files so Jekyll can include them.

## Technologies Used

- **Jekyll** with Minimal Mistakes theme: Static site generation
- **React** + **Vite**: Interactive frontend components
- **GitHub Pages**: Hosting

## Deployment

Pushing to the default branch triggers GitHub Pages to build and deploy the Jekyll site automatically. Before pushing, rebuild the React frontend and commit any changes to `assets/react/`:

```bash
cd frontend && npm run build
```
